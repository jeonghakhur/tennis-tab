🔧 최적화 상세 내역
1. getClubSessions — SELECT * 컬럼 최소화 + 불필요 attendance 컬럼 제거
세션 목록 조회 시 필요 없는 attendance의 session_id 컬럼 제거, sessions 테이블 명시 컬럼 지정
⚡ 성능 ▼ 45%
❌ Before — 1488ms
admin.from('club_sessions')
  .select('*, club_session_attendances(
    session_id, status, club_member_id
  )')
  .eq('club_id', clubId)
✅ After — 817ms
admin.from('club_sessions')
  .select('id, club_id, title, venue_name,
    court_numbers, session_date, start_time,
    end_time, max_attendees, status,
    rsvp_deadline, notes, created_by,
    created_at, updated_at,
    club_session_attendances(
      status, club_member_id
    )')
  .eq('club_id', clubId)
효과: attendance의 session_id 컬럼 제거 + sessions 전체 필드 명시로 전송 데이터량 감소. 1488ms → 817ms (671ms 절약)
2. tournaments/page.tsx — SELECT * → 7개 컬럼 + 유저 확인 병렬화
대회 목록 페이지에서 전체 컬럼 대신 TournamentCard에 필요한 7개만 선택, 유저 인증과 병렬 조회
⚡ 성능 ▼ 49%
❌ Before — 순차 + SELECT *
// 순차 실행 (getUser → getTournaments)
const { data: { user } } =
  await supabase.auth.getUser()

// 유저 확인 후 대회 조회
const { data: tournaments } =
  await supabase
    .from('tournaments')
    .select('*')  // 25+ 컬럼 전체
✅ After — 병렬 + 컬럼 최적화
// 유저 확인 + 대회 조회 병렬 실행
const [{ data: { user } }, { data: tournaments }]
  = await Promise.all([
    supabase.auth.getUser(),
    supabase.from('tournaments')
      .select('id, title, status,
        start_date, end_date,
        location, poster_url')
  ])
효과: 25+개 컬럼 → 7개 컬럼 (TournamentCard 실제 사용 컬럼만), 병렬화로 getUser 대기시간 제거. 337ms → 172ms (165ms 절약)
3. getPost — 조회수 업데이트 + 포스트 조회 병렬화
순차적으로 처리되던 view_count 조회 → 업데이트 → 포스트 조회를 병렬로 변경
⚡ 성능 ▼ ~33%
❌ Before — 3개 순차 쿼리
// 1) view_count 조회
const { data: current } = await admin
  .from('posts')
  .select('view_count').eq('id', id)

// 2) view_count 업데이트
if (current) {
  await admin.from('posts')
    .update({ view_count: current.view_count + 1 })
}

// 3) 포스트 상세 조회
const { data } = await admin
  .from('posts').select('*...')
✅ After — 병렬 처리
// 1+3 병렬 실행
const [{ data: current }, { data, error }]
  = await Promise.all([
    admin.from('posts')
      .select('view_count').eq('id', id),
    admin.from('posts')
      .select('*, author:profiles!...')
      .eq('id', id),
  ])

// 2) fire-and-forget
if (current) {
  admin.from('posts')
    .update({ view_count: current.view_count+1 })
    .then(() => {})
}
효과: 3개의 순차 쿼리를 2개 병렬로 축소. 포스트 상세 페이지 로딩 속도 약 33% 개선
