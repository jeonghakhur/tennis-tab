# Plan: 이용 안내 페이지 (Guide Page)

> **작성일**: 2026-03-03
> **버전**: 1.0

---

## 1. 개요

### 1.1 배경

Tennis Tab은 대회 참가, 클럽 활동, 자연어 AI 채팅이라는 세 가지 핵심 기능을 제공하지만, 처음 방문한 사용자가 각 기능을 어떻게 사용하는지 쉽게 파악할 수 있는 안내 페이지가 없다. 특히 메인 페이지가 AI 채팅 인터페이스로 구성되어 있어 서비스 전체 구조를 처음 접하는 사용자에게는 진입 장벽이 생긴다.

### 1.2 목표

- 3가지 핵심 기능(대회 참가, 클럽 이용, AI 자연어 검색)의 이용 방법을 단계별로 명확하게 안내
- 기능별 진입점(CTA)을 제공하여 안내 읽기 → 바로 실행으로 이어지는 흐름 구성
- 새 사용자의 이탈 감소 및 주요 기능 사용률 향상

### 1.3 범위

**In Scope:**
- `/guide` 단일 페이지 (3개 섹션: 대회 참가, 클럽 이용, AI 채팅)
- 각 섹션별 step-by-step 번호 안내 + 주요 포인트 강조
- 각 섹션 하단 CTA 버튼 (해당 페이지로 바로 이동)
- Footer에 '이용 안내' 링크 추가
- 섹션 간 앵커 링크 네비게이션

**Out of Scope (향후):**
- 온보딩 투어 (페이지별 툴팁 가이드)
- 역할별 분리 가이드 (클럽 임원, 대회 주최자 전용 페이지)
- 영상/GIF 튜토리얼
- 다국어 지원

## 2. 현재 서비스 구조 (분석 결과)

### 2.1 주요 사용자 흐름

```
[메인 페이지 /]         → AI 채팅 인터페이스 (ChatSection)
[대회 /tournaments]     → 목록 → 상세 → 신청 → 결제 → 대진표
[클럽 /clubs]           → 목록 → 상세(탭: 모임/순위/회원/입상) → 가입
[고객센터 /support]     → FAQ + 1:1 문의
```

### 2.2 사용자 역할

| 역할 | 설명 |
|------|------|
| 비로그인 | 대회/클럽 목록 조회 가능, AI 채팅 불가 |
| USER | 대회 참가 신청, 클럽 가입, AI 채팅 |
| 클럽 임원 (OWNER/ADMIN/MATCH_DIRECTOR) | 모임 생성/관리, 회원 관리, 대진 배정 |
| MANAGER 이상 | 대회 생성, 커뮤니티 글쓰기 |

## 3. 기능 요구사항

### 3.1 대회 참가 안내

**다루는 흐름**: 대회 찾기 → 상세 확인 → 참가 신청 → 결제 → 대진표 확인

**단계별 안내 내용:**

| 단계 | 제목 | 핵심 내용 |
|------|------|-----------|
| 1 | 대회 목록 확인 | `/tournaments`에서 상태별(접수중/예정/진행중) 대회 목록 확인 |
| 2 | 대회 상세 확인 | 일시·장소·참가비·부서 정보, 포스터, 대회 요강 확인 |
| 3 | 참가 신청 | 우측 사이드바의 '참가 신청' 버튼 → 부서 선택 → 정보 입력 |
| 4 | 결제 | 토스페이먼츠 온라인 결제 또는 계좌이체 |
| 5 | 대진표 확인 | 대진표 생성 후 '대진표 보기' 버튼 → 내 경기 일정·결과 확인 |

**주요 포인트:**
- 대회 상태 배지 의미 안내 (접수 예정 / 접수중 / 마감 / 진행중 / 종료)
- 로그인 필요 안내 (비로그인 상태에서 신청 버튼 클릭 시 로그인 페이지로 이동)
- 내 참가 내역 확인: 마이페이지(`/my/entries`)

**CTA**: `대회 목록 보기 →` (→ `/tournaments`)

---

### 3.2 클럽 이용 안내

**다루는 흐름**: 클럽 찾기 → 가입 → 모임 참석 → 경기 대진 → 순위 확인

**단계별 안내 내용:**

| 단계 | 제목 | 핵심 내용 |
|------|------|-----------|
| 1 | 클럽 찾기 | `/clubs`에서 이름 검색·지역 필터로 클럽 탐색 |
| 2 | 클럽 가입 | 가입 유형별 방법: 자유 가입(즉시), 승인제(자기소개 작성 후 대기), 초대 전용(초대 후) |
| 3 | 모임 참석 응답 | 클럽 상세 → 모임 탭 → 모임 선택 → 참석/불참/가능 시간 응답 |
| 4 | 경기 결과 확인 | 모임 마감 후 대진표 생성 → 내 경기 결과 및 점수 확인 |
| 5 | 순위 확인 | 클럽 상세 → 순위 탭 → 기간별·경기 방식별 순위 조회 |

**주요 포인트:**
- 클럽 탭 구성 안내: 모임(📅), 순위(📊), 회원(👥), 입상(🏆), 관리(⚙️·임원 전용)
- 가입 유형별 절차 차이 명확히 안내
- 임원(OWNER/ADMIN/MATCH_DIRECTOR)은 모임 생성/관리 권한 보유

**CTA**: `클럽 찾기 →` (→ `/clubs`)

---

### 3.3 AI 자연어 검색 안내

**다루는 흐름**: 메인 페이지 채팅창 → 자연어 입력 → AI 응답 → 필요 시 참가 신청/취소 플로우 진행

**단계별 안내 내용:**

| 단계 | 제목 | 핵심 내용 |
|------|------|-----------|
| 1 | 채팅창 접근 | 메인 페이지(`/`) → 하단 입력창에 원하는 것을 자유롭게 입력 |
| 2 | 대회 검색 | "지금 신청 가능한 대회 알려줘", "마포구 대회 있어?" 등 자연어로 검색 |
| 3 | 상세 조회 | "○○대회 참가비 얼마야?", "○○대회 대진표 보여줘" |
| 4 | 참가 신청 | "○○대회 신청하고 싶어" → AI가 단계별로 정보 수집 후 신청 |
| 5 | 참가 취소 | "신청 취소하고 싶어" → 신청 내역 확인 후 취소 |

**예시 질문 모음 (가이드에 표시):**

```
🔍 대회 검색
"지금 신청 가능한 대회 알려줘"
"다음 달 열리는 테니스 대회 있어?"
"마포 지역 대회 보여줘"

📋 상세 조회
"○○대회 요강 알려줘"
"내가 신청한 대회 목록 보여줘"
"다음 내 경기는 언제야?"

🏆 입상 기록
"최근 우승자 누구야?"
"올해 대회 입상 기록 보여줘"

📝 신청/취소
"○○대회 참가 신청하고 싶어"
"신청 취소하고 싶어"
```

**주요 포인트:**
- 로그인 필수 (비로그인 시 채팅 불가 → 로그인 유도)
- 참가 신청 플로우: AI가 대회 선택 → 부서 선택 → 전화번호 → 파트너(복식) → 확인 순으로 안내
- "취소"를 입력하면 진행 중인 플로우 중단 가능

**CTA**: `AI 채팅 시작 →` (→ `/`)

## 4. UI/UX 설계

### 4.1 페이지 구조

```
/guide
├── 페이지 헤더 (타이틀 + 간단한 소개)
├── 섹션 탐색 앵커 버튼 (3개: 대회 참가 | 클럽 이용 | AI 채팅)
├── Section 1: 대회 참가와 신청
│   ├── 섹션 제목 + 설명
│   ├── Step 1~5 (번호 + 아이콘 + 제목 + 설명)
│   └── CTA 버튼
├── Section 2: 클럽 이용
│   ├── 섹션 제목 + 설명
│   ├── Step 1~5 (번호 + 아이콘 + 제목 + 설명)
│   └── CTA 버튼
└── Section 3: AI 자연어 검색
    ├── 섹션 제목 + 설명
    ├── Step 1~5 (번호 + 아이콘 + 제목 + 설명)
    ├── 예시 질문 카드 그리드
    └── CTA 버튼
```

### 4.2 컴포넌트 구조

```
src/app/guide/
└── page.tsx               # 서버 컴포넌트 (정적 페이지)

src/components/guide/
├── GuideSection.tsx       # 섹션 래퍼 (제목 + 설명 + 슬롯)
├── GuideStep.tsx          # 단계 아이템 (번호 + 아이콘 + 제목 + 내용)
└── GuideExamples.tsx      # AI 채팅 예시 질문 카드 그리드
```

### 4.3 스타일 방향

- 기존 디자인 시스템 CSS 변수 (`--bg-primary`, `--text-primary` 등) 활용
- 스텝 번호: 강조 색상 원형 뱃지
- 섹션 구분: 수직 divider + 배경색 교차
- 예시 질문: `glass-card` 스타일 카드 그리드

### 4.4 접근성

- 각 섹션 앵커(`id="tournament"`, `id="club"`, `id="chat"`)로 직접 링크 공유 가능
- 스텝 아이콘에 `aria-hidden="true"` 적용
- CTA 버튼에 명확한 레이블
