# Plan: 자주하는 질문 (FAQ)

## 1. 개요

### 1.1 배경
Tennis Tab 서비스에는 1:1 문의 기능(`/support/inquiry`)은 있지만, 반복적으로 들어오는 공통 질문에 대한 자체 응답 시스템이 없다. 대회 참가 방법, 클럽 가입 절차, 점수 체계 등 동일한 질문이 1:1 문의로 반복 유입되어 관리자 부담이 가중되고, 사용자는 간단한 정보를 얻기 위해 불필요한 대기 시간을 겪게 된다.

### 1.2 목표
- 자주 묻는 질문을 카테고리별로 정리하여 사용자 자가 해결률 향상
- 1:1 문의 유입량 감소를 통한 관리자 업무 효율화
- 고객센터(`/support`) 페이지에 FAQ 섹션을 추가하여 지원 체계 완성

### 1.3 범위

**In Scope:**
- 카테고리별 FAQ 아코디언 UI (고객센터 페이지 내)
- 어드민에서 FAQ 항목 CRUD (생성, 수정, 삭제, 순서 변경)
- FAQ 검색 기능 (제목 + 내용 텍스트 검색)
- FAQ 카테고리 관리

**Out of Scope (향후):**
- FAQ 조회 통계 (인기 질문 순위)
- AI 기반 자동 답변 연동
- FAQ 항목 평가 (도움이 되었나요? 피드백)
- 다국어 지원

| 5 | 점수는 어떻게 입력하나요? | 마이페이지의 프로필 수정에서 본인의 점수를 직접 입력하실 수 있습니다. 점수 체계와 기준에 대해서는 커뮤니티의 대회 규칙 게시글을 참고해주세요. |

|---|------|------|

| 5 | 단체전과 개인전의 차이가 뭔가요? | 개인전은 1명(단식) 또는 2명(복식)이 한 팀으로 참가합니다. 단체전은 여러 명이 팀을 이루어 참가하며, 각 세트마다 출전 선수를 배정하는 방식으로 진행됩니다. |
| 6 | 예선과 본선은 어떻게 진행되나요? | 예선은 조별 풀리그 방식으로 진행되며, 각 조의 상위 팀이 본선에 진출합니다. 본선은 토너먼트(패자 탈락) 방식으로 진행되어 최종 우승자를 가립니다. |

## 4. 기능 상세

### 4.1 FAQ 카테고리 (동적 관리)

카테고리는 `faq_categories` 테이블에서 동적으로 관리된다. 어드민에서 CRUD 가능.

**초기 시드 데이터:**

| 카테고리 | slug | 표시 순서 |
|----------|------|-----------|
| 계정/인증 | `ACCOUNT` | 1 |
| 대회 | `TOURNAMENT` | 2 |
| 클럽 | `CLUB` | 3 |
| 커뮤니티 | `COMMUNITY` | 4 |

**카테고리 관리 규칙:**
- slug: 영문 대문자 + 숫자 + 언더스코어만 허용 (자동 정규화)
- 해당 카테고리에 FAQ가 있으면 삭제 불가 (FK 제약)
- 비활성화 시 사용자 화면에서 카테고리 탭 미표시

### 4.2 FAQ 데이터

- 카테고리 (필수)
- 질문 (필수, 최대 200자)
- 답변 (필수, 최대 2000자, HTML 지원)
- 표시 순서 (정수, 같은 카테고리 내 정렬)
- 활성화 여부 (비활성 FAQ는 사용자에게 미표시)

### 4.3 사용자 UI

- 고객센터(`/support`) 페이지 상단에 FAQ 섹션 배치
- 카테고리 탭 → 해당 카테고리 FAQ 아코디언 표시
- 검색 입력 시 전체 카테고리에서 질문+답변 텍스트 매칭
- 아코디언 열기/닫기 애니메이션
- FAQ 영역 하단에 "원하는 답변을 찾지 못하셨나요?" → 1:1 문의 유도

### 4.4 어드민 UI

- `/admin/faq` 페이지에서 FAQ CRUD
- 카테고리 필터 + 검색
- 인라인 순서 변경 (위/아래 버튼 또는 DnD)
- 활성화/비활성화 토글
- **카테고리 관리 모달**: 카테고리 추가/수정(이름, 정렬순서, 활성화)/삭제
  - 삭제 시 해당 카테고리에 FAQ가 있으면 거부 (서버 사이드 검증)

## 6. 데이터 모델

### 6.1 faq_categories (FAQ 카테고리)

| 필드 | 타입 | 제약 | 설명 |
|------|------|------|------|
| slug | text | PK | 영문 대문자 코드 (예: ACCOUNT) |
| name | text | NOT NULL | 카테고리 표시명 (예: 계정/인증) |
| sort_order | integer | NOT NULL, default 0 | 표시 순서 |
| is_active | boolean | NOT NULL, default true | 활성화 여부 |
| created_at | timestamptz | default now() | |

### 6.2 faqs (자주하는 질문)

| 필드 | 타입 | 제약 | 설명 |
|------|------|------|------|
| id | uuid | PK, default gen_random_uuid() | |
| category | text | NOT NULL, FK → faq_categories(slug) | 카테고리 slug |
| question | text | NOT NULL, max 200 | 질문 |
| answer | text | NOT NULL, max 2000 | 답변 (HTML 지원) |
| sort_order | integer | NOT NULL, default 0 | 카테고리 내 표시 순서 |
| is_active | boolean | NOT NULL, default true | 활성화 여부 |
| created_at | timestamptz | default now() | |
| updated_at | timestamptz | default now() | |

### 6.3 인덱스

- `idx_faqs_category_sort`: `(category, sort_order)` — 카테고리별 정렬 조회
- `idx_faqs_active`: `(is_active)` WHERE `is_active = true` — 활성 FAQ 필터

## 8. RLS 정책

### faq_categories
- SELECT: 전체 허용 (활성 카테고리만 공개)
- INSERT/UPDATE/DELETE: Server Action에서 ADMIN 권한 검증 (createAdminClient 사용)

### faqs
- SELECT: 전체 허용 (`is_active = true` 조건은 Server Action에서 처리)
- INSERT: ADMIN 이상
- UPDATE: ADMIN 이상
- DELETE: ADMIN 이상

## 9. 기술적 고려사항

### 9.1 Server Actions
- `src/lib/faq/actions.ts` — FAQ + 카테고리 CRUD
  - **카테고리**: `getFaqCategories()`, `getFaqCategoriesAdmin()`, `createFaqCategory()`, `updateFaqCategory()`, `deleteFaqCategory()`
  - **FAQ**: `getFaqs()`, `getFaqsAdmin()`, `createFaq()`, `updateFaq()`, `deleteFaq()`, `toggleFaqActive()`

### 9.2 타입 정의
- `src/lib/faq/types.ts` — FAQ, FaqCategory 타입

### 9.3 컴포넌트 구조
```
src/components/faq/
├── FaqSection.tsx          # 고객센터용 FAQ 영역 (카테고리 탭 + 아코디언)
├── FaqAccordion.tsx        # 아코디언 아이템 (질문/답변)
└── FaqSearch.tsx           # FAQ 검색 입력
```

### 9.4 고객센터 페이지 재구성
- 현재 `/support`는 카드 2개(문의, 내역)만 있는 단순 구조
- FAQ 섹션을 상단에 배치하고, 기존 1:1 문의 카드를 하단으로 이동
- 레이아웃: FAQ 영역 (전체 너비) → 구분선 → 문의 카드 (그리드)

### 9.5 성능
- FAQ 데이터는 적은 양(수십 개)이므로 전체 조회 후 클라이언트 필터링
- 검색도 클라이언트 사이드로 처리 (서버 부하 최소화)
- 초기 로딩은 Server Component에서 fetch하여 SSR

### 9.6 초기 데이터
- 2.1~2.5에서 정의한 FAQ 콘텐츠를 seed 스크립트로 일괄 등록
- `scripts/seed-faq.ts` 또는 마이그레이션 SQL에 포함
