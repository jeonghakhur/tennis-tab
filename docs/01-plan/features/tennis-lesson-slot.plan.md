# Plan: 테니스 레슨 슬롯 예약 시스템 (Lesson Slot Booking)

> 작성일: 2026-03-16
> 목적: 기존 레슨 프로그램 방식에서 **슬롯 기반 예약 방식**으로 전환
> 참고: `tennis-lesson.plan.md` (기존 플랜)

---

## 개요

코치가 레슨 가능한 시간 슬롯을 30분 단위로 미리 등록해두고,
회원이 빈 슬롯을 직접 선택해서 신청하는 방식.

코치가 특정 슬롯에 레슨자를 직접 배정하면 해당 슬롯은 잠기고,
나머지 빈 슬롯만 회원이 선택할 수 있다.

---

## 핵심 개념

```
코치 (coaches)
  └── 레슨 슬롯 (lesson_slots) ← 코치 또는 어드민이 미리 등록
        ├── OPEN     : 빈 슬롯 (회원 신청 가능)
        ├── LOCKED   : 코치/어드민이 특정 회원 직접 배정 (잠김)
        └── BOOKED   : 회원이 신청 완료 (잠김)

회원 (club_members)
  └── 레슨 예약 (lesson_bookings) ← 슬롯 선택 신청
        ├── 주 1회 : 슬롯 1개 선택
        └── 주 2회 : 슬롯 2개 선택 (주중/주말 혼합 가능)
```

## 주요 액터

| 액터 | 역할 |
|------|------|
| 클럽 어드민 | 슬롯 생성/삭제, 코치별 슬롯 관리, 회원 직접 배정 |
| 코치 (어드민 대리) | 어드민이 코치 대신 슬롯 및 배정 관리 |
| 클럽 회원 (로그인) | 빈 슬롯 1~2개 선택 후 신청 |
| 비회원 (비로그인) | 이름 + 연락처 입력 후 빈 슬롯 1~2개 선택 신청 가능 |

> **비회원 신청 정책:**
> - 로그인 없이 이름, 연락처(전화번호) 입력으로 신청 가능
> - 신청 완료 후 어드민에게 알림 발송
> - 어드민이 확인 후 수동 확정 (PENDING → CONFIRMED)
> - 비회원 신청은 `lesson_bookings` 테이블에 `guest_name`, `guest_phone` 컬럼으로 저장

| 요일 | 레슨 가능 시간 |
|------|--------------|
| 월, 화, 목 | 06:30 ~ 19:30 |
| 수, 금 | 06:30 ~ 14:00 |
| 토, 일 | 06:30 ~ 10:00 |

> 슬롯 등록 시 위 시간 범위 내에서만 등록 가능하도록 유효성 검사 적용.
> 코치별로 별도 가능 시간 설정도 가능하게 추후 확장 예정.

## 슬롯 등록 방식

### 코치/어드민이 레슨 슬롯 생성 (위저드)

1. **주당 회수**: 주 1회 / 주 2회 선택
2. **레슨 시간**: 20분 / 30분 선택
3. **요일**: 선택한 회수만큼 요일 선택 (주 2회면 2개)
4. **시작일**: 달력에서 첫 레슨 날짜 선택 (해당 월 말까지 슬롯 자동 생성)
5. **시간**: 요일별 시작 시간 설정
- 슬롯 생성 시 기본 상태: `OPEN`

> **시작일 기준 생성:** 시작일부터 해당 월 말일까지 선택 요일에 해당하는 날짜만 슬롯 생성.
> 월 중반에 시작하면 남은 횟수만큼만 생성 (4회 고정 아님).

```
예시: 주2회 — 화/목(30분), 시작일 3/18(화) 기준
→ 3/18(화) 10:00~10:30 OPEN
→ 3/20(목) 14:00~14:30 OPEN
→ 3/25(화) 10:00~10:30 OPEN
→ 3/27(목) 14:00~14:30 OPEN
(4개 생성 — 3월 18일부터 말일까지)
```

## 슬롯 상태

| 상태 | 설명 | 신청 가능 여부 |
|------|------|--------------|
| `OPEN` | 빈 슬롯 (회원 신청 가능) | ✅ 가능 |
| `BLOCKED` | 코치가 비공개 처리 (레슨 없음, 개인 사정 등) | ❌ 불가 |
| `LOCKED` | 코치/어드민이 특정 회원 직접 배정 | ❌ 불가 |
| `BOOKED` | 회원이 신청 완료 | ❌ 불가 |
| `CANCELLED` | 취소됨 (기존 예약 취소) | ❌ 불가 |

> **BLOCKED vs CANCELLED 차이:**
> - `BLOCKED`: 슬롯 자체를 열지 않음 (코치 휴무, 개인 사정 등) — 회원에게 노출 안 됨
> - `CANCELLED`: 기존에 OPEN/BOOKED 상태였다가 취소된 것 — 회원에게 취소 알림 발송

## 요금 설정

### 슬롯 수 × 주중/주말 조합별 요금

회원이 선택한 슬롯 수와 요일 조합에 따라 요금 자동 계산:

| 선택 | 요금 기준 |
|------|---------|
| 주 1회 (주중) | `fee_weekday_1` |
| 주 1회 (주말) | `fee_weekend_1` |
| 주 2회 (주중 2개) | `fee_weekday_2` |
| 주 2회 (주말 2개) | `fee_weekend_2` |
| 주 2회 (주중 1 + 주말 1) | `fee_mixed_2` |

> 요금은 **레슨 프로그램**에 설정하며, 슬롯 선택 시 자동 계산되어 표시됨.

### 요금 설정 규칙

- 최대 **2개 슬롯** 선택 가능
- 주중/주말 혼합 선택 가능
- 슬롯 선택 시 실시간으로 예상 요금 표시
- 실제 레슨 횟수는 시작일 기준으로 달라지므로 요금은 별도 협의 또는 회차당 요금 기준 적용

## 슬롯 신청 및 수락 플로우

```
[회원/비회원]
1. 코치 선택 → 빈 슬롯 달력 표시
2. 슬롯 선택 (최대 2개) + 요금 확인
3. 비회원: 이름/연락처 입력
4. 신청 완료
   └── 슬롯 상태: OPEN → BOOKED (임시 잠김)
   └── 예약 상태: PENDING
   └── 코치/어드민에게 수락 요청 알림 발송

[코치 / 어드민]
5. 수락 요청 알림 확인
6. 수락 → 예약 상태: PENDING → CONFIRMED
           슬롯 상태: BOOKED 유지 (확정 잠김)
           신청자에게 확정 알림 발송
   거절 → 예약 상태: PENDING → CANCELLED
           슬롯 상태: BOOKED → OPEN (슬롯 복구)
           신청자에게 거절 알림 발송
```

> **핵심:** 신청 시점에 슬롯은 임시로 잠기지만, 코치/어드민이 **수락해야 레슨 확정**.
> 거절 시 슬롯이 다시 OPEN으로 복구되어 다른 사람이 신청 가능.

## 레슨 확정 후 회원가입 유도

```
코치 수락 → CONFIRMED 알림 발송
    ↓
비회원에게 회원가입 안내 메시지 포함
"레슨이 확정되었습니다! 회원가입하시면
 레슨 일정 관리, 출석 확인 등을 앱에서
 편리하게 이용하실 수 있어요."
    ↓
회원가입 가능 → 기존 예약과 계정 연결
회원가입 어려운 경우 → 어드민이 수동 관리
  (예약 상태, 출석 등 어드민 페이지에서 직접 처리)
```

### 비회원 수동 관리 (어드민)

| 기능 | 설명 |
|------|------|
| 비회원 예약 목록 | 이름/연락처로 식별, 예약 현황 조회 |
| 출석 수동 체크 | 어드민이 직접 출석/결석 처리 |
| 메모 | 비회원별 특이사항 메모 기능 |
| 계정 연결 | 나중에 회원가입 시 기존 예약 이력 연결 가능 |
